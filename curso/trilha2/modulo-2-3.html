1

🔬 Anatomia do questions_v2

questions_v2 é uma tool que Claude chama. Ela NÃO retorna resposta imediata — encerra o turno para você responder. Por isso você vê Claude parar "aguardando".

Quando Claude invoca

  • • Projeto novo / ambíguo (ex: "faz um deck")
  • • Falta ponto de partida (sem DESIGN.md, sem repo)
  • • Pergunta genérica ("quero um site moderno")

Quando Claude PULA

  • • Ajustes pequenos num design existente
  • • Follow-up claro ("troque a fonte pra Geist")
  • • Contexto já foi dado no prompt inicial
  • • "recrie esta UI do codebase" — fonte pronta
2

📋 As 10 perguntas canônicas

O system prompt dita: "Faça pelo menos mais 4 perguntas específicas do problema" — ou seja, 10 é mínimo, não máximo. Acima você viu o mapa.

💡 Hack: pré-responda

Se você sabe essas 10 respostas de antemão, coloque tudo no prompt inicial → Claude pula completamente. É o que a skill design-designer (T5) faz.

3

🎯 Quando pular legitimamente

Pular não é sempre ruim. A decisão correta depende do contexto.

✓ Pular faz sentido

  • • Follow-ups em design existente
  • • Micro-ajustes visuais
  • • "recrie a UI desse repo" (fonte pronta)
  • • Prompt já front-loaded com tudo

✗ Pular = resultado pior

  • • Projeto do zero
  • • Sem DESIGN.md
  • • Pedido genérico ("algo bonito")
  • • Nenhuma referência visual
4

⚡ Override: user prompt > system prompt

System prompt pede "pelo menos 10 perguntas". User prompt ganha prioridade. Você pode pedir menos, nenhuma, ou N específicas.

Frases que funcionam

// Pedir poucas perguntas
"Me faça só as 3 perguntas mais essenciais."

// Pular completamente (pouco recomendado)
"Skip questions. Generate using defaults."

// Focar perguntas específicas
"Pergunte apenas sobre handoff final e Tweaks.
 O resto use meu DESIGN.md."

// Responder antecipadamente
"Audiência: solo founders técnicos. Feeling: late-night
 cinematic. 5 variações (2 cor + 2 type + 1 layout radical).
 Handoff: Claude Code. Sem orbes flutuantes."
5

✍️ Como responder: curto e específico

2-5 palavras por resposta. Se não sabe, "escolha você" é válido — Claude toma decisão informada baseada no que já viu. Respostas longas consomem tokens sem melhorar qualidade.

🔄 Pre-answer: a skill design-designer

A skill oficial design-designer faz entrevista OFFLINE (antes de você abrir o Claude Design) e entrega prompt pronto para colar — cobrindo todas as 10 perguntas.
