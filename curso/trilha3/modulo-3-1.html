📜 Anatomia do system prompt (~10.000 palavras)

1

📜 Identidade: "designer-manager"

A primeira linha define tudo: "You are an expert designer working with the user as a manager. You produce design artifacts on behalf of the user using HTML."

Implicações práticas

  • • Claude NÃO questiona suas decisões — executa
  • • Foco em delivery, não em debate conceitual
  • • Você é o "gerente" — mudanças de direção são aceitas sem pushback
  • • Meio de saída: HTML sempre, mas o formato varia (deck, landing, protótipo, animação)
2

📰 Viés newspaper embutido: a anti-slop list

System prompt lista explicitamente o que evitar — e é justamente isso que aparece quando o usuário sub-especifica.

🚫 A lista oficial do anti-slop

  • ×Gradientes agressivos (mesh, blob) como plano de fundo
  • ×Emojis (exceto se o DS os usa)
  • ×Containers arredondados com accent de borda esquerda
  • ×Imagens desenhadas em SVG (usar placeholders)
  • ×Fontes superusadas: Inter, Roboto, Arial, Fraunces
3

❓ "Make at least 10 questions" — texto literal

Não é sugestão, é instrução: "Faça pelo menos 10 perguntas, talvez mais". Daí vem a sensação de que Claude Design "pergunta demais".

⚡ Override

User prompt > system prompt. "Pule questions_v2 e use estes defaults: [X, Y, Z]" funciona. Mas só use com contexto — sem info, output cai.

4

🪜 Escada rígida de 3 variações

O prompt diz: "Try to give 3+ variations. Start your variations basic and get more advanced and creative as you go."

Implicação

Sem override: você recebe exatamente 3 — safe, balanced, novel. Para mais, peça explícito e nomeie axis (ver T2 módulo 2.6).

5

🧠 Workflow em 6 passos (pipeline interno)

Pipeline fixo que Claude executa:

  1. Entender necessidades + perguntas (questions_v2)
  2. Explorar recursos (DESIGN.md, repo, URLs, arquivos)
  3. Planejar (todo list interno)
  4. Construir estrutura de pastas + arquivos
  5. Finalizar: done() → se erros, corrige → fork_verifier_agent
  6. Resumir extremamente brevemente — só ressalvas e próximos passos
6

🔒 Regra de IP (email-domain gate)

Uma proteção de propriedade intelectual embutida: Claude recusa recriar padrões de UI distintivos, estruturas proprietárias ou elementos visuais com marca de outra empresa.

⚖️ Como funciona

Claude verifica o domínio do seu email:

  • ✓email @stripe.com pedindo "recrie a UI do Stripe" — permitido
  • ✗email @anyone.com pedindo "recrie a UI do Stripe" — recusa + oferece alternativa original
