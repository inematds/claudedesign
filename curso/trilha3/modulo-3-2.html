🎁 Os 7 starters e quando usar

1

📊 deck_stage.js — o pilar dos decks

Obrigatório por system prompt para QUALQUER apresentação. Web component que envolve slides como <deck-stage> com <section> como filhos diretos.

O que vem embutido

  • • Escala 1920×1080 via transform:scale() + letterbox preto
  • • Navegação: teclado (←/→/Space), touch, wheel
  • • Overlay de contagem "03/12" no canto
  • • postMessage({slideIndexChanged: N}) para speaker notes
  • • Persistência de posição via localStorage
  • • Print-to-PDF: uma página por slide
  • • Auto-rotula [data-screen-label="01", "02"...] e [data-om-validate]
2

🎨 design_canvas.jsx — grid de variações

Para apresentar 2+ opções estáticas lado a lado. Células rotuladas, scroll-snap opcional, sizing responsivo.

Quando usar vs quando evitar

  • ✓Use: comparar variações visuais puras (cor, layout, tipografia estática)
  • ✗Evite: variações de flow/interação — melhor protótipo clicável com Tweak toggle
3

📱 ios_frame / android_frame

Molduras mobile realistas. O iOS inclui notch, status bar, home indicator, teclado virtual. Android tem status + nav bar. O "realismo percebido" do protótipo sobe ~80%.

💡 Combinação poderosa

design_canvas com 3 células + ios_frame dentro de cada = 3 flows mobile lado a lado. Ideal para comparar variações de onboarding.

4

🖥️ macos_window / browser_window

macos_window: traffic lights + título. Para SaaS desktop, apps CLI, ferramentas "de profissional". browser_window: address bar + tabs + back/forward. Para mostrar landing "vista no browser real".

Hack

Quando você apresenta landing pra cliente, envolver em browser_window faz o feedback ficar 3x melhor — cliente não questiona "e o browser?" e foca no design.

5

🎬 animations.jsx — timeline engine

Coberto em detalhe no módulo 3.5. Resumo: Stage + Sprite + useTime/useSprite + Easing + interpolate. Use SEMPRE antes de cair em Popmotion (regra do system prompt).

6

🧩 copy_starter_component: a tool

Tool que Claude chama com kind: "deck_stage.js". Retorna código pronto que você pode importar via <script src> (JS) ou <script type="text/babel" src> (JSX).

⚠️ Extensão é crítica

Tool falha se você passar "deck_stage" (sem .js) ou "deck_stage.jsx" (extensão errada). Sempre exata.

// ✓ certos
copy_starter_component({kind: "deck_stage.js"})
copy_starter_component({kind: "ios_frame.jsx"})

// ✗ errados
copy_starter_component({kind: "deck_stage"})      // falha
copy_starter_component({kind: "ios_frame.js"})   // falha
