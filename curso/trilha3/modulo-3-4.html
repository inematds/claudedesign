🔬 O que é window.claude.complete

Função JavaScript nativa disponível dentro de qualquer HTML gerado pelo Claude Design. Chama um LLM sem SDK, sem API key, sem configuração. É o que permite protótipos com AI real embutida.

Por que importa

Protótipos com interação real (não só visual): dashboard que explica dados, chatbot funcional, sugestão de próximos passos, classificação de input do usuário. Sem setup.

2

⚠️ Modelo: Haiku 4.5 (NÃO Opus)

Segredo que o system prompt leak revelou: apesar de o mock "dizer" que usa Opus 4.7 nos bastidores, a função realmente chama claude-haiku-4-5.

🧠 Implicação de capacidade

  • ✓Bom em: summarização curta, classificação, extração, resposta simples
  • ✗Ruim em: raciocínio multi-step, código complexo, análise profunda
  • ✓Ótimo: <2s de latência, baixo custo operacional
3

🔢 Cap de 1024 tokens (FIXO)

Max 1024 tokens de output. Não há parâmetro para aumentar. Não há workaround. Dimensione seus prompts com isso em mente.

1024 tokens = aprox

FormatoCapacidade
Texto corrido inglês~750 palavras
Texto corrido português~650 palavras
Código JS~50 linhas
JSON estruturado~30-40 chaves
Bullets curtos~15-20 itens
4

💻 API: duas formas de chamar

Forma simples e forma com estrutura de mensagens:

// Forma 1: string direta
const text = await window.claude.complete("Explique em 3 bullets: ...");

// Forma 2: messages (com system prompt customizado + histórico)
const text2 = await window.claude.complete({
  messages: [
    { role: 'system', content: 'Você é um analista financeiro.' },
    { role: 'user', content: 'Analise este dado: ...' }
  ]
});

// Encadeamento (histórico simulado)
const text3 = await window.claude.complete({
  messages: [
    { role: 'user', content: 'Olá' },
    { role: 'assistant', content: 'Oi! Como posso ajudar?' },
    { role: 'user', content: 'Me fale sobre ...' }
  ]
});
5

🚦 Rate limit por viewer

Cada usuário que abre o mock consome quota sua. Compartilhou via URL? Cada visitante = chamadas contadas na SUA conta.

⚠️ Cuidado em demos ao vivo

Apresentação com 30 pessoas? Se todos interagirem com o mock, você pode zerar quota em 5 minutos. Estratégia: cache resposta ou chame só em click explícito.

6

🎯 Use cases ideais

Dado os limites (Haiku + 1024 tokens + rate limit), o sweet spot é:

✓ Use para

  • • Dashboard com insights dinâmicos ("explique esse número")
  • • Chatbot simples (3-5 turnos)
  • • Classificação (este feedback é positivo/neutro/negativo?)
  • • Sugestão de próximos passos
  • • Extração de entidades (nome, data, valor)
  • • Summarização curta

✗ Evite para

  • • Geração de texto longo (blog, email corrido)
  • • Código complexo (> 50 linhas)
  • • Análise profunda multi-step
  • • Conversa longa (> 10 turnos)
  • • Produção real (use a API de verdade)

✅ Resumo do Módulo

✓
API nativa no mock — sem SDK, sem key, só window.claude.complete().
✓
Haiku 4.5 (não Opus) — apesar de dizer outras coisas nos bastidores.
✓
Cap de 1024 tokens fixo — ~650 palavras, ~50 linhas de código, ~20 bullets.
✓
Duas formas de chamar — string simples ou messages com histórico.
✓
Rate limit por viewer — cuidado em demos ao vivo.
✓
Sweet spot — insights, sugestões, classificação. Não gere texto longo aqui.
