🔒 Verificador silencioso (fork_verifier_agent)

O subagente que revisa com olhos frescos em contexto isolado. Mesma filosofia do Claude Code: nunca deixar o modelo revisar o próprio trabalho.

🔄 Fluxo do verificador: do done() à confirmação

1

🪞 Por que um agente separado?

Princípio vindo do Claude Code: nunca deixar o modelo revisar o próprio trabalho no mesmo contexto. O contexto acumulado contamina a avaliação. Subagente novo = olhos frescos.

Analogia

Imagine você escreveu um texto. Reler 30s depois: você vê o que quis dizer, não o que escreveu. Reler 24h depois (ou pedir pra colega ler): você vê os erros de verdade. O verificador é o "colega".

2

🎯 Quando roda automaticamente

Trigger: após done() retornar clean (sem erros de console). Claude invoca fork_verifier_agent() sem argumentos.

Pipeline completo

  1. Claude termina design
  2. Chama done(path) — abre preview
  3. Se erros de console → Claude corrige e chama done() de novo
  4. Se clean → invoca fork_verifier_agent()
  5. Verificador roda em background (não bloqueia)
  6. Claude pode encerrar turno
3

🔍 O que o verificador checa

Iframe próprio com 4 tipos de check:

📸 SCREENSHOTS

Múltiplos viewports (desktop, tablet, mobile). Compara com expectativa visual.

📐 LAYOUT PROBE

Overflow horizontal, elementos sobrepostos, spacing quebrado, alinhamento.

⚙️ JS SONDAGEM

Console errors, promises rejeitadas, event handlers quebrados.

👁️ VISUAL CHECK

Contraste ruim, texto cortado, imagens não carregadas.

4

🎯 Verificação direcionada

Você pode pedir um check focado MID-task — não só no final:

// No meio da iteração, peça:
"Tire um screenshot e confira se o espaçamento entre
 os 3 cards está consistente em mobile."

// Claude invoca:
fork_verifier_agent({
  task: "Check spacing consistency between 3 cards in mobile viewport"
})

// Verificador volta independentemente com o report.
// Você não precisa de done() antes — só para delivery final.
5

⛔ Não espere pelo verificador

System prompt: "Não espere por ele; encerre seu turno." Verificador é assíncrono. Claude continua, você continua. Se der problema, ele interrompe com relato.

💡 Na prática

Você vê mensagem "design pronto, pode revisar". Se 30s depois aparecer "⚠️ Verificador detectou: button cortado em 375px viewport" — é o verificador. Até lá, está tudo certo.

6

🚫 NÃO faça self-check

System prompt é explícito: "Não faça sua própria verificação antes de chamar done; não capture screenshots proativamente para revisar seu trabalho; conte com o verificador."

🚨 Por que é regra

Self-check polui o contexto do Claude com detalhes (screenshots, probes, console outputs) que não importam pro usuário. Role separation: Claude CRIA, verificador CHECA.
