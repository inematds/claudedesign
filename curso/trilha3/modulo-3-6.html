🗺️ Mapa das 6 pegadinhas

1

📰 Newspaper default sub-especificado

Sintomas: italic serif, off-white, pull-quote editorial, layout "Our Portfolio". Fix: DESIGN.md + referências explícitas + restrições ("sem italic serif, sem off-white").

2

📐 Decks são 1920×1080 fixo

deck_stage.js hardcoded 16:9. Para 9:16 (stories, Instagram Reels), 4:3 (legacy), 1:1 (square), peça explicitamente ANTES de pedir slides. Senão retrabalho.

"Crie slides 9:16 (vertical, 1080×1920) para stories.
 NÃO use deck_stage.js default — faça manual com scale
 próprio. Stage de viewport height."
3

🚫 Sem tela de "título" em protótipos

Regra intencional do system prompt: "Resist the urge to add a title screen." Se quer uma tela inicial de "splash/título", peça explicitamente.

4

🔒 Cross-project read-only

Claude pode LER arquivos de outros projetos, mas assets (imagens, CSS) não carregam via path externo. Peça copy_files para copiar para o projeto atual antes de referenciar.

5

⛔ scrollIntoView proibido

System prompt: "Never use scrollIntoView — it can mess up the web app." Quebra o iframe do Claude Design. Use window.scrollTo() ou element.offsetTop + manual scroll.

6

🏷️ Style objects com nome único (NON-NEGOCIÁVEL)

Babel scripts inline compartilham escopo global. Se dois componentes têm const styles = {}, o segundo sobrescreve o primeiro — crash silencioso.

// ✗ ERRADO — crasha
const styles = { button: {...} };  // em button.jsx
const styles = { card: {...} };    // em card.jsx → SOBREWRITE

// ✓ CERTO
const buttonStyles = { ... };
const cardStyles = { ... };
// Ou use inline styles
