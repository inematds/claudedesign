🤖 Skill design-designer (entrevista offline)

Skill Claude Code que entrevista você localmente com as 10 perguntas canônicas — entrega prompt pronto para colar no Claude Design.

1

🎯 O que é a skill design-designer

Skill oficial Claude Code que faz a entrevista do Claude Design offline, no seu terminal. Você responde 10 perguntas. Ela entrega prompt pronto para colar. Resultado: zero perguntas em claude.ai/design.

Como invocar

$ claude
> /design-designer

ou apenas: "build me a claude design prompt for a landing"
2

❓ As 10 perguntas da skill

Cobrem exatamente o que Claude Design perguntaria — mas offline, onde você tem tempo para pensar:

  1. Q1: Tipo (landing, deck, proto, app clicável, website, dashboard)
  2. Q2: Produto + audiência em 1 frase
  3. Q3: Feeling dominante + secundários
  4. Q4: Contexto para importar (Figma, repo, screenshots)
  5. Q5: Brand colors e tipografia
  6. Q6: Quantas variações e axis
  7. Q7: Sections/screens específicas
  8. Q8: O que NÃO deve ter (kill biases)
  9. Q9: Output e handoff
  10. Q10: Tweaks a expor no canvas
3

⚡ Output: bloco copy-paste

A skill entrega prompt estruturado que cobre todas as 10 perguntas + restrições explícitas contra AI tropes + tweaks nomeados + handoff. Cola no claude.ai/design e a geração começa imediatamente.

4

🛠️ Customizando a skill

Edite ~/.claude/skills/design-designer/SKILL.md para adicionar perguntas específicas da sua vertical (ex: "saúde → perguntas sobre compliance, LGPD, trust signals").

5

🎬 Quick path: skip interview

Se você já sabe tudo: "skip the interview, just build it" + brief curto. Skill infere defaults sensatos e avisa o que você não especificou — você pode overridar na hora.

6

📚 Combo skill + DESIGN.md

Combo mágico: skill referencia DESIGN.md do projeto atual. Output já vem com brand system amarrado. Zero setup, output premium.
