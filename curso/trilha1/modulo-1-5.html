♾️ O loop que substitui "prompt perfeito"

1

🔁 O loop: motor mental

Três fases, uma só máquina: Gerar → Refinar → Exportar. O refinar volta para o gerar se você descobrir que precisa mudar estrutura. Nunca é linear. Aceite isso e você para de travar.

🎯 A regra dos 80%

Dos seus 100% de tempo na ferramenta, ~15% vai para o gerar inicial, ~80% para refinar, ~5% para exportar. Se você está gastando mais de 30% tentando ajustar o prompt perfeito, está fazendo errado.

2

🎯 Prompt ruim + refinamento = win

O hack contra-intuitivo mais importante: um prompt simples + 10 ajustes pequenos frequentemente BATE um prompt de 500 palavras "perfeito" de primeira.

✗ Overthinking

Você gasta 15 min escrevendo o prompt "definitivo", com detalhes de cor, tipografia, spacing, copy. Quando gera, está quase certo — mas não exatamente.

Custo: 15 min de planejamento + 5 min de geração + retrabalho porque faltou um detalhe.

✓ Seed + iteração

Você escreve "landing pra [produto] pra [audiência], feeling [X]". 30s. Claude gera. Você ajusta via Tweaks e comments.

Custo: 30s de prompt + 2 min de geração + 5 min de refinamento. Total: menos da metade.

3

💬 Comentários específicos > reclamações vagas

"O design está estranho" é a pior coisa que você pode dizer. Claude não sabe o que fazer. Seja cirúrgico.

✗ Vago

  • "não gostei"
  • "pode ser mais moderno?"
  • "tá genérico"
  • "melhor o hero"

✓ Cirúrgico (comment no elemento + texto)

  • [no botão CTA] "contraste fraco, deixe o fundo em #0ea5e9"
  • [no hero] "headline com no máximo 8 palavras, estilo Linear"
  • [nas 3 cards] "alinhe à esquerda e reduza padding pela metade"
  • [no footer] "remova, vou pôr manualmente"
4

🎛️ Tweaks > chat para micro-ajustes

Sempre que a mudança é quantitativa (cor X, tamanho Y, densidade Z), vá pro painel Tweaks. Chat só para mudanças estruturais.

Árvore de decisão

Tipo de mudançaOnde fazer
Cor, tamanho, densidadeTweaks
Trocar fonte de um conjunto conhecidoTweaks
Reescrever copy de um elementoInline edit (duplo-clique)
Mover um elemento específicoComment
Adicionar seção novaChat
Mudar estética geralChat + re-gerar
5

✏️ Inline edit: o atalho pra copy

Duplo-clique em qualquer texto no canvas — Claude Design entra em modo de edição de texto direto. Você edita e salva. Sem passar pelo chat, sem consumir tokens.

⚡ Velocidade relativa

  • •Inline edit: ~3 segundos por mudança
  • •Comment com texto novo: ~15 segundos (mais iteração do Claude)
  • •Chat: ~30-60 segundos (Claude pensa, gera, atualiza)
6

🚪 Quando parar: a curva de retornos

Overfitting visual é real. Se você iterou 15 vezes e está empatando ou piorando — pare. Exporte.

📉 Curva de retornos decrescentes

🛑 Sinais de que é hora de parar

  • ×Os últimos 3 ajustes empataram ou pioraram
  • ×Você está alternando entre duas versões sem preferência clara
  • ×Mostrou para alguém e a pessoa reagiu positivamente — é sinal de sweet spot
  • ×Passou 30+ minutos ininterruptos — perdeu perspectiva, olhe amanhã

✅ Resumo do Módulo

✓
Loop não-linear — gerar → refinar → exportar, com loops internos. 80% do tempo no refinar.
✓
Seed simples > prompt perfeito — 30s de prompt + iteração bate 15 min de planejamento.
✓
Comentários cirúrgicos — marque elemento + texto específico. Vago = travado.
✓
Árvore de decisão — Tweaks pra quantitativo, inline edit pra copy, chat pra estrutura.
✓
Sweet spot ≈ 5-7 iterações — depois disso, retornos decrescentes e risco de overfitting.
✓
Sinais de parar — 3 ajustes empatados, alternância sem preferência, validação externa positiva.
