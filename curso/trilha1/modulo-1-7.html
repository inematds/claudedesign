✅ Checklist do iniciante

Os 10 erros mais comuns de quem abre o Claude Design pela primeira vez — e como corrigir cada um em segundos. Vira reflexo após 2-3 sessões.

📋 Cheatsheet visual: os 10 erros em um olhar

1

❌ Erro 1: Prompt sem marca → newspaper default

O viés newspaper-minimalist está dentro do system prompt. Se você não fornece design system, Claude cai nele automaticamente: italic serif, off-white, pull-quotes, layout editorial. Todo design fica parecido.

✗ Sintoma

Você gerou 3 designs completamente diferentes mas todos parecem jornal — serif, off-white, seção "Our Portfolio" editorial.

✓ Correção em 30s

Anexe (1) 2-3 hex codes da marca + (2) URL de referência (competidor ou site que você ama). Isso já quebra 90% do viés.

2

❌ Erro 2: Pular as perguntas iniciais

A tentação é grande: Claude pergunta 10+ coisas, você quer ver o design logo. Resista. 90s respondendo isso melhora o resultado dramaticamente.

⚡ Correção: responda rápido mas responda

Respostas curtas (2-5 palavras por pergunta) são ideais. Se não sabe, diga "não sei, escolha você" — Claude trata como carta branca. NUNCA diga "skip" ou "just generate".

3

❌ Erro 3: Re-promptar em vez de Tweak

Mudar cor, tamanho ou densidade via chat é 10x mais caro em tokens (e tempo) que mexer em um slider. Não faça.

Regra mental

Antes de digitar no chat, pergunte: "isso é QUANTITATIVO ou ESTRUTURAL?"

  • • Quantitativo (mais X, menos Y, cor Z) → Tweaks
  • • Estrutural (adicione seção, mude fluxo, troque stack) → Chat
4

❌ Erro 4: Aceitar só 3 variações

O system prompt estipula 3 variações por padrão. Para ter 5, 6 ou 7, você PRECISA pedir explicitamente — e nomear o axis.

Frase pronta

"Me dê 6 variações explorando 3 axis:
- 2 de paleta (mais quente vs mais frio)
- 2 de tipografia (serif editorial vs sans geométrico)
- 2 de densidade (arejado vs compacto)
Start safe, progress to novel."
5

❌ Erro 5: Colar conteúdo em vez de referenciar

Colar 10 páginas de briefing no chat vs referenciar o arquivo: a diferença é gigantesca. Path reference economiza 50-70% de tokens pela mesma qualidade de output.

✗ Colar

[cola 8000 palavras do briefing no chat]

Essas 8000 palavras ficam no contexto a SESSÃO INTEIRA — mesmo depois que você mudou de assunto.

✓ Referenciar

"Leia briefing.pdf e gere a landing"

Claude lê seletivamente, usa snip automático, mantém contexto enxuto.

6

❌ Erros 6 a 10: os outros clássicos

Os últimos 5 que completam o checklist — cobrem 95% das frustrações iniciais juntos com os 5 anteriores.

6

Zero referência visual

"Moderno, limpo, profissional" não é estética — é eu-não-sei-o-que-quero disfarçado.

Correção: anexe 2-3 URLs de sites cujo feeling você admira. Ou nomeie uma família estética (Editorial Minimalism, Cinematic Dark, Terminal-Core).

7

Cross-project errado (read-only esquecido)

Você referencia arquivo de OUTRO projeto Claude Design — ele lê, mas o HTML renderizado não consegue carregar a imagem/asset porque o path é externo.

Correção: se vai USAR (não só ler) asset de outro projeto, peça "copy this asset into the current project".

8

Ignorar o verificador silencioso

fork_verifier_agent roda em background. Se ele flagrar algo, você recebe mensagem. Ignorar = bug passa batido.

Correção: sempre leia o output final do verificador antes de exportar. Se flagar console error, peça "corrija os erros do verificador".

9

Esquecer quem é a audiência

"Landing page de produtividade" gera copy genérico (produtivo = bom?). Sem audiência específica, Claude não tem como calibrar tom.

Correção: audiência precisa ser comportamental, não demográfica. "Solo founders que odeiam notificação" > "homens 25-40".

10

Export no formato errado

Exportou PDF para passar pro dev (ele queria HTML). Exportou HTML para apresentar (cliente queria slides). Retrabalho.

Correção: decida o export ANTES de começar — faz parte da prep de 30s do módulo 1.4. Deck pra dev → Claude Code. Pra cliente → PDF ou Share URL.
